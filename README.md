# База знаний проекта «GigaSpec» (Sber AI Hack / Ouroboros)

Единая точка входа в контекст проекта. Все документы самодостаточны и рассчитаны на
использование в отдельных сессиях — можно подключать по одному под конкретную задачу.

## Что это за проект в двух словах

**Самоэволюционирующийся SDD-оркестратор поверх Ouroboros (аналог Claude Code) и MCP Atlassian.**
Не описываем процесс регламентом — **добываем его из команды: интервью + скан её живых
источников**, затем ведём любую задачу
по этапам конвейера команды (готовность/ТЗ → аналитика → дизайн → код) до PR, охраняем входящие,
учим новичков и **эволюционируем** по сигналам из практики. Коммитимся на полный цикл идея→PR;
каждый этап — с апрувом человека и чек-поинтом. Один источник правды — `.md`-файлы.

Формула: **Добывает → Применяет → Охраняет → Учит → Эволюционирует.**

### Зафиксированные решения (синк 02–04.07; открытое — [DISCUSSION.md](DISCUSSION.md), протокол — в git-истории)

- **Имя продукта — GigaSpec** (ранее «Ready»); сабмит-имя проекта — «Giga Uroboros SDD Orchestrator».
- **Коммитимся на полный цикл идея→PR**; звезда демо — весь конвейер **и** фаза ретро/самоэволюции.
- **Деньги/финмодель сняты** — продаём качество и предсказуемость, не ROI (замер AS IS — архив).
- **Ядро MVP (§3):** самоэволюция, калибровка-интервью, MCP Atlassian+Telegram, полный SDD,
  чек-поинты, дашборд+мониторинг, классификатор задачи.
- **UI-конструктор из блоков** — отдельная ветка (Ваня), в гарантированный скоуп не входит.
- **Каналы связи — агностичны**; дефолт аудируемости — комментарий в Jira + пинг автора;
  **для демо — Jira + Telegram**.
