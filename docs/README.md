# База знаний проекта «Ready» (Sber AI Hack / Ouroboros)

Единая точка входа в контекст проекта. Все документы самодостаточны и рассчитаны на
использование в отдельных сессиях — можно подключать по одному под конкретную задачу.

## Что это за проект в двух словах

**Самоэволюционирующийся SDD-оркестратор поверх Ouroboros (аналог Claude Code) и MCP Atlassian.**
Не описываем процесс регламентом — **добываем его из примеров задач**, затем ведём любую задачу
по этапам конвейера команды (готовность/ТЗ → аналитика → дизайн → код) до PR, охраняем входящие,
учим новичков и **эволюционируем** по сигналам из практики. Каждый этап — самостоятельная
ценность: конвейер можно остановить на готовом ТЗ. Один источник правды — `.md`-файлы.

Формула: **Добывает → Применяет → Охраняет → Учит → Эволюционирует.**

### Зафиксированные решения (02.07)

- **Заявка продаёт конвейер идея→PR целиком**; прототип гарантирует этапы до готового ТЗ +
  охрану + эволюцию, этап кода — ранняя версия (happy-path до PR), скипается без потери ценности.
- **Конструктор из блоков** — целевое видение, вне MVP (настройка через `.md`).
- **Каналы связи — агностичны** (вход из любого канала, выход в любой транспорт/MCP Ouroboros);
  дефолт аудируемости — комментарий в Jira-задаче + пинг автора; **для демо — Jira + Telegram**.
