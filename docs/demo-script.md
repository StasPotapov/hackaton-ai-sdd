# Демо-сценарий (~3 минуты)

> То, что выигрывает. Магия настоящая, но прогон отрепетирован — никакого «всё вживую без сетки».

## Хронометраж

| Время | Сцена | Что показываем |
|---|---|---|
| **0:00** | Проблема | «Все хотят SDD. Никто не знает как. Внедрение = ноль.» → сырые задачи, «а что ты имел в виду?». |
| **0:20** | Калибровка + интервью | Скилл берёт свою базу «хорошего процесса» + 5 реальных задач жюри → **доспрашивает** пару вопросов → выдаёт **черновик ИХ стандарта**. *«Мы не принесли вам процесс. Сейчас достанем ваш.»* |
| **0:55** | Фича 1 — заводит задачу сам | Обсуждение в чате / встреча из **Jazz** → тег агента → **готовая задача в Jira по процессу**. *«Обсуждение больше не теряется.»* |
| **1:30** | Фича 2 — охраняет входящие | Кто-то завёл сырую задачу → скилл **сам комментирует**: «нет критериев приёмки, нет макета — докиньте». |
| **2:05** | Превращение | Сырая задача → процесс готовности → готовое ТЗ + ссылка в Confluence, метка **Ready**. Вживую. |
| **2:40 → 3:05** | Фича 3 — самоэволюция | Подпись «через 2 недели» → дайджест: «8 задач вернули из-за отсутствия метрик — предлагаю добавить пункт в DoR» → `diff` к `.md` → тимлид жмёт **Apply**. Финальный кадр со схемой 5 глаголов и конвейером: *«Сегодня — до готовности. Тот же конвейер уже умеет вести дальше — до PR на зелёном quality gate»* (+ кадр happy-path PR, если этап кода дозрел). |

## Killer-моменты (на них держится показ)

1. **«Мы не принесли вам процесс — сейчас достанем ваш»** — калибровка из 5 задач + интервью.
2. **Заводит задачу сам** из чата/Jazz — «обсуждение → задача в один тег».
3. **Охраняет входящие** — сам комментирует, чего не хватает.
4. **Самоэволюция** — финал: улучшает процесс команды через `diff` к `.md`.

## Страховка (антириск на сцене)

- **Сид данных заранее** — реальные задачи подготовлены, не собираем вживую с нуля.
- **Записанный фолбэк в кармане** — на случай сбоя интеграций/сети.
- Прогон **отрепетирован** по секундам.

## Замечания по критериям Project Results

Демо-видео = 30% оценки основного этапа, «результаты на примерах» = 20%. Значит показ должен:
- содержать **конкретные примеры** (реальные задачи, реальный `diff`, реальные метрики);
- явно демонстрировать **применение Ouroboros** (10%);
- быть чистым по качеству материалов (10%).
