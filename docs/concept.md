# Концепция: 5 глаголов и единый движок

> Продуктовая суть. Что делает скилл и почему это цельная система, а не набор фич.

## Пять глаголов — нервная система продукта

> **Добывает → Применяет → Охраняет → Учит → Эволюционирует.**
> Один источник правды (`.md`), 4 контура работы + петля обратной связи. Замкнутый цикл.

| Глагол | Контур | Что делает |
|---|---|---|
| **Добывает** | Калибровка | Интервью с командой + скан её источников (Jira, Confluence, чаты) → выводит неявный стандарт команды → пишет чек-лист и шаблоны в `.md` |
| **Применяет** | Конвейер (Ready Loop + этапы) | Проводит задачу по этапам конвейера: петля готовности до порога readiness score → дальше по процессу команды вплоть до кода и PR |
| **Охраняет** | Охрана | Мониторит очередь, комментирует, ловит дубли, возвращает недооформленное |
| **Учит** | Онбординг | Отвечает новичку «как у нас принято?» на реальных примерах команды |
| **Эволюционирует** | Эволюция (петля) | Копит сигналы со всех контуров → предлагает `diff` к `.md` → человек апрувит |

## Единый движок — все роли

Меняется только чек-лист «что значит готово», механика одна:

| Роль | Вход | Выход |
|---|---|---|
| Аналитик | сырая гипотеза | ТЗ передаётся в разработку без вопросов |
| Продакт | идея фичи | PRD согласован, метрики и scope зафиксированы |
| Дизайнер | задача на макет | бриф + критерии для проектирования |
| Разработчик | готовое ТЗ | PR на зелёном quality gate (этап кода конвейера) |

Это ключевой ров: **один примитив, много ролей** — продаёт менеджменту, не только разработке.

## Движок из 4 шагов (контур «Применяет» детально)

1. **Калибровка через интервью** (один раз на команду). Скилл не с нуля — в него заложена база
   «каким должен быть хороший процесс». Главный источник — **интервью**: скилл расспрашивает
   команду, как опытный тимлид на онбординге. Параллельно **сканирует все доступные источники**
   (бэклог Jira, Confluence, чаты) — как свидетельства практики, не эталон: сверяет «как говорим»
   с «как делаем», расхождения превращает в вопросы. Команде без истории хватает базы + интервью.
   Итог → чек-лист + шаблоны в `.md`. Команда правит файлы — это вся настройка, без кода и UI.
2. **Захват.** Тянет задачу из Jira через MCP Atlassian, подтягивает контекст: связанные задачи,
   эпик, похожие прошлые.
3. **Процесс готовности.** Черновик ТЗ → **самокритика по чек-листу** → уточняющие вопросы
   человеку, если данных не хватает → готовое ТЗ.
4. **Выгрузка и следующие этапы.** Готовое ТЗ/PRD в Confluence, дозаполнение Jira, метка
   **Ready**. Дальше задача идёт по этапам процесса команды — вплоть до кода в изолированной
   ветке и PR на зелёном quality gate. Конвейер останавливается на любом этапе.

## База — это мнение, а не пустой шаблон

Жёсткий формат, мягкое содержание. Мы не даём «гибкую базу» (= пустоту), мы даём **конкретный
чек-лист с позицией**. Команда спорит с конкретным → в споре находит свой процесс. Это
принципиально отличает от «дайте нам конструктор, заполните сами».

## Три главные фичи (killer, на них давим)

1. **Заводит задачи сам, по вашему процессу.** Тегни агента в чате или дай встречу из **Jazz** →
   он вытащит суть из обсуждения/расшифровки и откроет готовую задачу в Jira по вашему стандарту.
   Обсуждение → задача в один тег.
2. **Охраняет входящие.** Мониторит очередь Jira; в заведённой задаче сверяет с процессом и, если
   чего-то не хватает (описание, макет, критерии, метрика), **сам комментирует автору** и просит
   докинуть. Недооформленное не идёт дальше.
3. **Самоэволюция процесса** (то, что хотят жюри). Раз в спринт смотрит на задачи команды — что
   влетало, как менялось, как часто переоткрывалось — и **предлагает, чем улучшить процесс**
   (`diff` к `.md` → апрув человека).

Опорные месседжи демо: **«Мы не принесли вам процесс — сейчас найдём ваш»** (калибровка:
интервью + скан источников) и превращение сырой задачи в готовое ТЗ.

## Схема петли (эволюция поверх всех контуров)

```
            checklist.md / template.md
                       ▲
                       │ apply (после approve)
                       │
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
    Захват  Готовность  Охрана  Онбординг
        │       │          │        │
        └───────┴─ сигналы ─┴────────┘
                       │
                       ▼
              ┌─ Эволюция ─┐
              │ дайджест + │
              │  diff к .md│
              └────────────┘
                       │
                       ▼
                Human review
```
