## Принцип

Не полноценное исследование, а **замер по выборке с честной пометкой источника**:
«по данным команды X за последние 6 месяцев, выборка N задач». Для жюри одна реальная цифра
весит больше десяти допущений.

## Шаг 0. Выбрать рамку (5 минут)

- Проект(ы) Jira своей команды: `___`
- Период: последние **6 месяцев** (или 2 полных квартала).
- Тип задач: Story/Task (баги можно исключить или считать отдельно).

## Шаг 1. Объём задач (10 минут) → параметр «число задач в год»

JQL в поиске Jira:

```
project = XXX AND issuetype in (Story, Task) AND created >= -26w
```

- Записать **общее число** → годовой объём = ×2.
- Экспортировать CSV (key, summary, created, resolved, status) — пригодится для шагов ниже.

## Шаг 2. Доля возвратов (15 минут) → параметр «число возвратов в год»

Основной вариант:

```
project = XXX AND issuetype in (Story, Task) AND created >= -26w AND status was Reopened
```

Если статуса Reopened нет — прокси «уехала назад по доске»:

```
project = XXX AND created >= -26w AND status changed from ("In Progress", "In Review") to ("Backlog", "To Do", "Open")
```

- Доля возвратов = результат / итог шага 1.
- ⚠️ Названия статусов подставить свои (посмотреть на доске).

## Шаг 3. Время до готовности (30 минут) → метрика «время до Ready»

Точный статус Ready есть редко, поэтому **прокси: от создания до первого перехода в
In Progress** (всё это время задача «дозревала»).

- **Вариант А (руками, надёжно):** взять 20 случайных задач из CSV шага 1 → в каждой открыть
  History → записать дату создания и дату первого перехода в In Progress → медиана по 20.
- **Вариант Б (скрипт):** если есть API-токен Jira — принести мне base URL + токен, я напишу
  скрипт по `/rest/api/2/search?expand=changelog` и посчитаю по всей выборке автоматически.

## Шаг 4. Итерации уточнений (30 минут) → метрика «кол-во итераций»

По тем же 20 задачам из шага 3: посчитать в комментариях **число уточняющих вопросов до начала
работы** (вопросы «а что имелось в виду», запросы макета/критериев и т.п.). Записать среднее.
Заодно пометить 2–3 самых показательных примера — пригодятся для деки и демо.

## Шаг 5. Параметры финмодели (15 минут, без Jira)

- **Ставка часа (fully loaded):** средняя месячная ЗП роли × 1,5 (налоги + накладные) / 168 ч.
- **Число новичков в год** в команде/подразделении: `___` (спросить тимлида/HR).
- **Δчасов на груминг задачи** и **Δчасов на переделку возврата** — экспертная оценка команды
  по 3–5 знакомым задачам (честно пометить «экспертная оценка, N=5»).
